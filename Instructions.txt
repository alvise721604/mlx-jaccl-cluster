INIZIO:

Su entrambi i Mac:
git clone https://github.com/alvise721604/mlx-jaccl-cluster.git
	cd mlx-jaccl-cluster

conda create -n mlx python=3.14 -y
conda activate mlx
python -m pip install --upgrade pip wheel setuptools
python -m pip install mlx mlx-lm huggingface_hub transformers fastapi uvicorn

VERIFICA:

	python - <<'PY'
	import mlx.core as mx
	import mlx_lm
	print("mlx ok")
	print("default device:", mx.default_device())
	PY

Deve stampare:

	mlx ok
	default device: Device(gpu, 0)

Setup for macstudio1
====================
sudo ifconfig bridge0 down
sudo ifconfig en3 inet 192.168.0.1 netmask 255.255.255.252
sudo route change 192.168.0.2 -interface en3

TEST DOWNLOAD di un Model:
Su entrambi i Mac:
mkdir -p ~/models_mlx
hf download mlx-community/Qwen3-4B-Instruct-2507-4bit  --local-dir ~/models_mlx/Qwen3-4B-Instruct-2507-4bit

Su entrambi i Mac:
Modificare path python alla prima riga dello scripts/python-mlxjccl in accordo col reale path del python 3.14 corrente
sudo cp scripts/python-mlxjccl /usr/local/bin/

Sul primo Mac:
sudo mkdir -p /usr/local/etc/
sudo cp hostfiles/hosts_mac_cluster.json /usr/local/etc/hosts.json

Su secondo Mac fare un sistema di symlinks in modo che sia visibile /Volumes/Home/dorigo_a come sul primo Mac

Dal primo Mac provare inferenza distribuita:

	mlx.launch \
	  --backend jaccl \
	  --hostfile /usr/local/etc/hosts.json \
	  --cwd /Volumes/Home/dorigo_a/mlx-jaccl-cluster \
	  --python /usr/local/bin/python-mlxjccl \
	  --env MLX_METAL_FAST_SYNCH=1 \
	  --env HF_HUB_OFFLINE=1 \
	  --env TRANSFORMERS_OFFLINE=1 -- \
	  /Volumes/Home/dorigo_a/mlx-jaccl-cluster/scripts/jaccl_tps_bench.py \
	  --model /Volumes/Home/dorigo_a/models_mlx/gpt-oss-20b \
	  --prompt "Write 5 sentences about Thunderbolt RDMA." \
	  --max-tokens 1024

Ora lanciare dal primo Mac:

./launch_llm_engine.sh # che blocca la console in quanto attiva un server permanente.
